Philosophy

Built for GitOps teams

Most dashboards let you edit resources directly. Klarity deliberately does not — because in a proper GitOps workflow your cluster's source of truth lives in Git, not a web form.

Klarity fills the gap that GitOps creates: you need complete visibility into what's running, real-time diagnostics when something goes wrong, and a way to share cluster state with your team — all without the risk of accidental mutations bypassing your review and pipeline process.

Features

Everything you need to observe your cluster

Complete visibility without the risk of accidental mutations.

Auto-Discovery

Calls GET /apis on startup and maps every resource type — including all installed CRDs. Istio, Cert-Manager, ArgoCD appear automatically.

Live Log Streaming

Real-time WebSocket logs with severity filter (ERROR/WARN/INFO/DEBUG), keyword search, auto-scroll, and download. Pinned sessions survive navigation.

Web Terminal

Full kubectl exec experience via xterm.js with auto-reconnect. No local kubectl required — works from any browser.

GitOps Integration

Auto-detects ArgoCD and Flux CD by scanning cluster CRDs. Sync status, health, reconciliation, AppSets, HelmReleases — all visible, zero configuration.

User Management

Built-in auth with bcrypt passwords, three roles (admin/editor/viewer), 5-attempt lockout, 8h JWT tokens, and full OIDC/SSO support.

Real-time Metrics

CPU and Memory via metrics-server. Node circular gauges, container-level breakdown, top consumers, namespace summaries, and visual progress bars.

Port Forwarding

HTTP proxy tunnel via SPDY — access any pod's service port directly in your browser without running kubectl locally.

Audit Log

Every operation recorded in a 2000-event ring buffer — user, action, resource, namespace, HTTP status. Downloadable cluster health HTML reports.

Read-Only by Design

No create/edit/delete for cluster resources. Your cluster's source of truth is Git, not a UI form. Safe to grant broad team access.

Installation

Deploy in 30 seconds

Works with any Kubernetes 1.26+ cluster — kind, minikube, EKS, GKE, AKS, k3s.

Recommended for production 
# Add the Helm repository
helm repo add klarity https://selvarajmurugesan90.github.io/klarity
helm repo update

# Install Klarity
helm upgrade --install klarity klarity/klarity \
  --namespace klarity \
  --create-namespace \
  --set config.authMode=internal

# Access the dashboard
kubectl port-forward svc/klarity 8080:8080 -n klarity

Open http://localhost:8080 — Default: admin / admin@123 (change required on first login)

Quick install via Kustomize 
# Deploy with kubectl
kubectl apply -k https://github.com/selvarajmurugesan90/klarity/deploy/manifests

# Access the dashboard
kubectl port-forward svc/klarity 8080:8080 -n klarity

Open http://localhost:8080 — Default: admin / admin@123

GHCR image or Docker Compose 
# Pull the image
docker pull ghcr.io/selvarajmurugesan90/klarity:latest

# Or run with Docker Compose (local dev)
git clone https://github.com/selvarajmurugesan90/klarity
cd klarity && docker compose up

Mounts ~/.kube/config read-only. Open http://localhost:8080
